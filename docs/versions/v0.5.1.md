# v0.5.1 Release Notes

Release date: February 28, 2026

## What Changed
- Added a full commerce-loop E2E smoke test harness
- Added deterministic E2E mode behavior (cleanup scheduler disabled)

## How To Run Smoke Test
1. Start backend + database (docker or local)
2. Enable E2E mode in backend

```bash
E2E_MODE=true pnpm --filter ideaapp-backend dev
```

3. Run the smoke test:

```bash
pnpm --filter ideaapp-backend e2e:smoke
```

### Against Docker
If running via docker-compose, ensure backend is reachable at `http://localhost:7101`, then run:

```bash
API_BASE_URL=http://localhost:7101 pnpm --filter ideaapp-backend e2e:smoke
```

## Known Limitations
- Uses synthetic accounts and mock payment proof
- Requires E2E_MODE enabled to allow admin registration

## QA Checklist
- Backend running with `E2E_MODE=true`
- Database reachable and migrated
- `/api/v1/healthz` returns 200
- `pnpm --filter ideaapp-backend e2e:smoke` passes

## Troubleshooting
- If auth fails, clear conflicting test accounts in the DB or adjust test emails in `backend/scripts/e2e-smoke.js`.
- If sync pending is empty, verify merchant token and confirm proposal acceptance created a SyncRequest.
- If delivery draft not found, ensure order was marked `READY_FOR_DELIVERY` after `PAYMENT_VERIFIED`.
- If admin login fails, confirm `E2E_MODE=true` so ADMIN registration is allowed.
